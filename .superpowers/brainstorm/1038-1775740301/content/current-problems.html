当前 Workflow 面板的问题

基于代码分析，以下是两个核心可用性问题

问题 1：Reviewer 参与不可见

当前 StepCard 显示的 agents 列表是静态定义的：

sample ● running
├ clef-composer (sonnet)
├ clef-harmonist (sonnet)
└ clef-rhythmist (sonnet)
缺失：
+ clef-reviewer ← melody gate 审查
+ clef-reviewer ← full review
iterate ● running
├ clef-reviewer (sonnet)
└ clef-revision (sonnet)
缺失：
+ clef-leader ← 任务调度决策

PHASES 中的 agents 是静态列表，不反映运行时实际参与的 agent

问题 2：长时间运行无进度信息

create phase 内部有 5+ 个子步骤，但用户只看到：

create ● running
已运行 2 分 30 秒...
没有其他信息

实际 orchestrator 内部流程：

1. 生成和声 (clef-harmonist) ← 可能 30-60s
2. 生成旋律 (clef-composer) ← 可能 30-60s
3. 生成节奏 (clef-rhythmist) ← 可能 30-60s
4. 合并声部 (merge_abc)
5. 技术验证 (validate_abc)
6. 修正失败声部 (可能重试)
7. 转换 MIDI (abc_to_midi)
→ 总计可能 3-5 分钟，全程黑盒

改进方向选择

你希望优先解决哪个层面的问题？

A

两者都要

修复 agents 显示 + 添加子步骤进度（推荐，完整解决可用性问题）

B

只修 agents 显示

在 PHASES 静态列表中补全 reviewer/leader，不改后端进度机制

C

只做子步骤进度

实现后端 sub-step 事件 + 前端进度条，agents 显示保持现状
