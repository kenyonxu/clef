A: 缩进子步骤列表

在当前 StepCard 内部展开子步骤，用 ✓/→/○ 表示完成/进行中/待执行。改动最小，信息密度高。

Step create — 完整创作 ● running
生成节奏 (clef-rhythmist) 3/6
预计剩余 ~1m 30s

B: 进度条 + 当前子步骤

显示总进度条和当前正在执行的子步骤名称。简洁但信息量较少，子步骤历史不可回溯。

12:03:21 生成和声 done
12:04:08 生成旋律 done
12:04:52 生成节奏 running

C: 时间线日志

每个子步骤带时间戳，以日志流形式展示。可回溯历史，但占用垂直空间较多。
